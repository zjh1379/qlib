## 1. 背景与动机

### 1.1 研究目标（用户）
以**研究者心态**迭代选股/预测模型，在各层面突破现有基线，**以扣费后净收益为优先**，最看重 **1–5 日持仓**的短期高收益（低买高卖）。日内数据**先不投入**，作为第二阶段杠杆。产出需同时服务**选股 + 择时**。

### 1.2 现有基线
- 模型：LGBM(Alpha158) + ALSTM(Alpha360) + TRA(Alpha360)，各跑 1d/5d/20d，Ridge stacking → rank-avg 兜底 → EWMA(α=0.5)。
- 数据：仅日线，baostock，2018-01 → 2026-05（~2035 交易日），CSI800(~843) + ETF。
- 标签：开盘→开盘 `Ref($open,-(N+1))/Ref($open,-1)-1`（次日开盘买，N 日后开盘卖）——设计良好，贴合散户成交。
- 自定义因子：**0 个**，全靠标准 Alpha158/360。

### 1.3 关键发现（驱动本设计）
1. **现 `metrics.py` 的"回测"是信号代理，非真回测。** `_portfolio_returns` 每日重选 top-K，并把**整段 N 日收益**当作"当日收益"再 `×252` 年化。对 5d 标签 = 重叠计算 + 约 5× 高估，换手按"每日全换"算。**IC/ICIR 可信**（对 N 日标签的相关性），但**收益/IR/换手/年化全部失真**——所谓"年化 54%"即此产物。
2. **换手 46–49%/天**与用户"1–5 天持仓"完全矛盾；当前评估实为日频调仓。对齐持仓周期可砍 70–80% 换手。
3. **月多空差 ≈ 0.21%** 几乎为 0，说明区分能力弱；用更长可信窗口看，真实信号约 **IC≈0.026、ICIR≈0.20**（弱但真实）。
4. **小资金 + 最低 5 元手续费**使绝对手续费占比极高，必须显式建模（见 §3.2）。

**结论**：先建可信、可落地的净收益评估，再降换手、再叠短线因子。模型架构突破与专用择时留后续阶段。

## 3. P0 — 成本感知回测引擎（核心）

### 3.1 持仓模拟器（修掉重叠计算）
**分数与 P&L 解耦**：
- 输入 (a) 模型分数：**仅用于排序**；(b) 真实**日频开盘→开盘 1 日收益**（来自 qlib 价格）：用于逐日记 P&L。
- **不再**用 N 日标签当日收益。
- 逐日流程：用 ≤T-1 的分数定目标组合 → T 开盘成交 → 持有期内每天按 1 日收益记净值 → **仅在实际换仓日**按 `|Δ权重|` 对应的**绝对成交金额**计费 → 从日净值曲线正确年化。
- **无前视**：T 日决策只用 ≤T-1 信息。

模块：`production/backtest/engine.py::PortfolioBacktest`
- 状态：`positions: dict[instrument, weight]`、`nav: float`、`equity_curve`。
- 产出：日净值序列、日换手序列、每笔成交流水（用于成本对账与调试）。

### 3.2 成本模型（含最低费 + 资金量感知）★关键细化
模块：`production/backtest/costs.py::CostModel`

| 项 | 默认（profile=small） | 方向 | 说明 |
|---|---|---|---|
| 佣金率 | 2.5 bp（万2.5） | 双边 | |
| **佣金最低** | **¥5.0** | 每笔 | 小资金下常主导成本 |
| 印花税 | 5 bp（0.05%） | **仅卖出** | |
| 过户费 | 0.1 bp | 双边 | 沪市，含即可 |
| 滑点 | 5 bp | 双边 | CSI800 流动性较好；小盘可调高 |

- **单笔成本** = `max(notional × commission_bps, commission_min_yuan)` + 卖出印花税 + 过户费 + 滑点。
- `notional` 由**总资金 `capital`** 与目标权重推出 → 故 `capital` 是回测参数。
- 未来 profile=`pro`：`commission_bps=1.0, commission_min_yuan=0.0`（"万1免5"），通过配置切换，**预留接口**。
- **资金量敏感性分析**：在 {¥50k, ¥100k, ¥300k, ¥1M} 下分别评估，输出"该资金量的最优 top_k / 持仓周期"。

### 3.3 多 regime 长窗口评估
- 复用现有 regime 划分（2018 熊 / 2019–20 / COVID / 高波动 / AI 行情 / 近端）。
- 每段 + 全窗各出一张净收益记分卡。
- **评估数据**：为得到可信多 regime 数字，P0 含一次**有界的 LGBM 预测回填**（目标 2019→2026，**仅 LGBM**，在 `safety_watchdog` 下运行，控算力防卡死）。神经模型仍在近端窗口评估。

### 3.4 净收益指标
模块：`production/backtest/metrics_net.py`
- **净 CAGR、净 IR、净 Sharpe、最大回撤、真实换手、胜率、平均持有期收益、成本占比**。
- **复用** `metrics.py` 的 IC/RIC/ICIR/regime_split（信号质量部分本就正确）；**替换**其组合收益模拟。

## 4. P1 — 降换手组合构建

### 4.1 再平衡策略（实测对比，按净收益挑最优）
模块：`production/backtest/rebalance.py::RebalancePolicy`

| 策略 | 规则 | 作用 |
|---|---|---|
| `Daily(top_k)` | 每日重选 top-K | 复现诚实基线 |
| `FixedPeriod(H, top_k)` | 每 H 天重组 | 换手 ≈ 基线/H |
| **`Banded(enter_k, exit_k)`** | rank ≤ enter_k 买入；rank > exit_k 才卖出 | **换手杀手**：既粘住又响应。默认主力策略 |

### 4.2 持仓周期扫描
- 用户：1–5 天均合理。扫 **H ∈ {1, 2, 3, 5}**，按净收益（含最低费）选最优；默认 H=5。

### 4.3 中性化
模块：`production/neutralize.py`
- 对分数做**行业/市值中性**（行业内排序或回归剔除），减少无意暴露、提升净 IR。可配置开关。
- **数据依赖**：行业分类需补 baostock `query_stock_industry` 映射（小脚本，纳入计划）。市值用成分股市值或 `vwap×volume` 代理。

### 4.4 小资金/最低费下的组合分析 ★细化
- 在最低 ¥5 + 小 `capital` 约束下，联合扫 **top_k ∈ {5,10,15,20,30}** × **H ∈ {1,2,3,5}** × 策略，输出净收益前沿。
- 预期结论：小资金favor**更少只数 + 更低换手**；Banded + 小 top_k 价值进一步放大。明确给用户"该持几只、多久换"。

## 5. P2起步 — 短线因子库

### 5.1 因子取舍分析（避免与 Alpha158 冗余）
LGBM 已用 Alpha158，往里加冗余因子≈0 收益。剔除如下（已被 Alpha158 覆盖）：

| 候选 | Alpha158 等价 | 裁决 |
|---|---|---|
| 日内动量 close/open | KMID | ❌ |
| 振幅 (high-low) | KLEN | ❌ |
| 短期反转 −ROC5 | ROC5 | ❌ |
| 量价背离 corr | CORR/CORD | ❌ |
| 量能突增 vol/MA | VMA | ❌ |
| 上下影线 | KUP/KLOW/KSFT | ❌ |
| 日内位置 | RSV（窗口版） | ⚠️ 弃 |

### 5.2 入选因子（非冗余、有短线 α）

| 因子 | 定义要点 | 入选理由 |
|---|---|---|
| **隔夜跳空** | `$open/Ref($close,1) - 1`（及其 5d 均值/波动） | Alpha158 无"今开 vs 昨收"；跳空衰竭/反转强信号 |
| **隔夜/日内收益分解** | overnight = open/prev_close；intraday = close/open | 两者在 A 股符号不同，KMID 仅含日内 |
| **距涨停/触板频率** | board-aware 涨跌停（主板10%/创业科创20%），近窗触板次数 | Alpha158 无涨跌停概念；连板/触板是 A 股特异关注度信号，复用 Tier1 `parse_board` |
| **换手率 + 换手异动** | `turn`，`turn/MA(turn,N)` | Alpha158 仅 volume 比率，无真实换手率（÷流通股）；换手率是 A 股核心关注度因子 |
| **特异波动率/残差动量**(stretch) | 对市场回归的残差波动/动量 | RESI 是对时间回归非对市场；低特异波动 α。复杂度高，时间允许才做 |

剔除：**资金流/主力净流入**——日线做不好，推迟到 P4 日内。

### 5.3 数据依赖（扩展抓取）
- baostock 日线接口本就返回 `amount`(成交额) 与 `turn`(换手率)，现仅取 OHLCV。
- **扩展** `production/incremental_refresh.py::fetch_one` 字段为含 `amount,turn`，相应扩展 CSV 列与 `dump_bin`，并回填历史这两个字段。
- 解锁：换手率、换手异动、`amount`/`vwap×volume` 资金代理。

### 5.4 集成
- 新 handler `production/factors/handler.py::AlphaShortTerm = Alpha158 + 短线因子块`，先喂 **LGBM**（表格路径）。
- 神经网络（Alpha360）注入短线因子留 **P3**。

## 6. 数据流与集成
1. 扩展数据抓取（amount/turn）→ dump 至 qlib bin。
2. 补行业分类映射。
3. `AlphaShortTerm` handler 生成增强特征 → 训练/回填 LGBM 预测（有界、watchdog 下）。
4. `PortfolioBacktest`（成本+策略+中性化）消费"分数 + 真实日收益"→ 净收益记分卡（per regime + 全窗 + 资金敏感性）。
5. `production/backtest/run.py` CLI：单 recorder 评估 / 诚实基线 vs 改进对比 → `production/reports/*.json`。

---

## 7. 测试策略
- 🔴 **无前视测试（最关键）**：每个因子/分数在日期 d 只用 ≤d 数据（类比已有 walk-forward 不重叠测试）。
- **成本数学**：佣金/最低费/印花税(仅卖)/滑点逐项核对；不同 `capital` 下最低费触发正确。
- **换手单调性**：断言 Banded/FixedPeriod 换手 < Daily。
- **净值对账**：净收益 = 毛收益 − 成本；曲线连续性、与流水一致。
- **合成数据 golden test**：已知答案的小样本端到端。
- **重叠修复回归**：对同一分数，新引擎对 1d 标签结果应与"诚实日频"一致；对 5d 不再出现 ×5 高估。

---

## 8. 验收标准
1. 同一成本模型下，改进组合（Banded + 因子 + 中性化）**扣费净 IR** 显著 > 诚实基线（Daily）：日净收益序列**配对 t 检验 p<0.10**，或**净 IR 提升 ≥ 20%**。
2. **真实换手**较 Daily 基线下降 **≥ 50%**。
3. **各 regime 净 IR > 0**。
4. 无前视测试与成本测试**全过**。
5. 小资金（¥100k）+ 最低 5 元成本下，改进组合**净 CAGR > 0** 且 > 诚实基线。
6. 因子增量：加入短线因子后净 IR / IC 较"仅 Alpha158"有**可测提升**（报告 delta；薄信号不强设阈值）。

## 12. 如何服务"选股 + 择时"
- **选股**：成本感知 + 中性化的排序器 → 净收益验证过的 top-K 候选池。
- **择时（本期雏形）**：Banded 策略即"按排名持续性决定进出"的择时原语；隔夜跳空 / 距涨停 / 换手异动为**入场择时**信号。
- **完整择时**：专用持有/退出头（P3）。
