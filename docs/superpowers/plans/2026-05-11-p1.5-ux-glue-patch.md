### Endpoints

```
GET  /api/data/status
     Returns: {
       calendar_end: "2026-05-08",
       calendar_size: 2022,
       instruments_count: 300,
       last_refresh_at: "2026-05-11T11:23:45+08:00" | null,   // mtime of calendars/day.txt
       freshness: "fresh" | "stale_1d" | "stale_2d_plus",
     }

GET  /api/instruments?market=csi300
     Returns: {
       market: "csi300",
       count: 300,
       items: [
         { symbol: "SH600519", name: "贵州茅台" },
         { symbol: "SH600036", name: "招商银行" },
         ...
       ]
     }
     Reads CN names from production/cn_names_cache.json (already exists from akshare).

POST /api/data/refresh
     Triggers update_qlib_data.ps1 as a detached subprocess.
     Returns: { job_id: "uuid", started_at: "...", message: "Refresh started." }
     Implementation: subprocess.Popen with PowerShell; non-blocking;
     write job entry to logs/ with stdout/stderr captured.
     If a refresh is already in progress (check by lock file), return 409.

GET  /api/data/refresh/{job_id}
     Returns: { job_id, status: "running"|"done"|"failed", started_at, finished_at, log_tail }
```

## Acceptance criteria

- [ ] `GET /api/data/status` returns valid JSON with calendar_end + freshness
- [ ] `GET /api/instruments?market=csi300` returns 300 items with Chinese names (95%+ coverage)
- [ ] `POST /api/data/refresh` spawns the .ps1 successfully (validated by checking PID exists, not by waiting for completion)
- [ ] Dashboard at `/` shows system status + data card + symbol search + popular links
- [ ] Search box in nav works on every page; "茅台" → finds SH600519
- [ ] Clicking a popular ticker navigates to its chart
- [ ] All previous tests still pass (35 backend, 2 frontend → expect 40+ backend, 3+ frontend)
